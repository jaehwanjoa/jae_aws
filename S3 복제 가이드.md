# S3 로그 복제 가이드

크로스 어카운트 구조에서 S3 복제 규칙을 통해 로그를 복제하는 방안을 가이드합니다.

---

## 1. IAM 정책 생성(원본 로그 계정에서 수행)
AWS CloudShell에서 아래와 같이 Trust Policy를 생성합니다.

```bash
cat <<EOF > trust-policy.json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Service": "s3.amazonaws.com"
            },
            "Action": "sts:AssumeRole"
        }
    ]
}
EOF
```
IAM Role 생성하고 Trust Policy를 적용합니다.
```bash
aws iam create-role \
  --role-name cross-account-bucket-replication-role \
  --assume-role-policy-document file://trust-policy.json
```
IAM Role에 ONS-S3-Replicate-Policy 생성
```bash
cat <<EOF > ONS-S3-Replicate-Policy.json
{
    "Version": "2012-10-17",
    "Statement": [
    {
        "Sid": "SourceBucketPermissions",
        "Effect": "Allow",
        "Action": [
        "s3:ListBucket",
        "s3:GetReplicationConfiguration",
        "s3:GetBucketVersioning",
        "s3:GetBucketLocation",
        "s3:GetObjectVersionForReplication",
        "s3:GetObjectVersionAcl",
        "s3:GetObjectVersionTagging"
        ],
        "Resource": [
        "arn:aws:s3:::aws-waf-logs-cjl-collector-an2",
        "arn:aws:s3:::aws-waf-logs-cjl-collector-an2/*",
        "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1",
        "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1/*"
        ]
    },
    {
        "Sid": "DestinationBucketReplicationWrite",
        "Effect": "Allow",
        "Action": [
        "s3:ReplicateObject",
        "s3:ReplicateDelete",
        "s3:ReplicateTags",
        "s3:PutObject",
        "s3:PutObjectAcl",
        "s3:ObjectOwnerOverrideToBucketOwner"
        ],
        "Resource": [
        "arn:aws:s3:::aws-waf-logs-cjl-an2",
        "arn:aws:s3:::aws-waf-logs-cjl-an2/*",
        "arn:aws:s3:::aws-waf-logs-cjl-ue1",
        "arn:aws:s3:::aws-waf-logs-cjl-ue1/*"
        ]
    }
    ]
}
EOF
```
IAM Role에 Permission Policy 연결

```bash
aws iam put-role-policy \
  --role-name cross-account-bucket-replication-role \
  --policy-name AllowPutEvents \
  --policy-document file://ONS-S3-Replicate-Policy.json
```
## 2. S3에서 복제 규칙 생성(원본 로그 계정에서 수행)
원본 S3 버킷을 선택합니다. 아래와 같이 설정합니다.
```bash
§ 속성: 버킷 버전 관리 활성화 적용
§ 관리 > 복제 규칙 생성
    □ 이름: s3-replication-rule
    □ 규칙범위: 버킷의 모든 객체에 적용
    □ 대상: 다른 계정의 버킷 지정
        ® 계정 ID: {AccountID}
        ® 버킷 이름: {S3 버킷 이름}
    □ IAM 역할: setp 4에서 생성한 역할 적용
    추가 복제 옵션: 복제 시간 제어(RTC) 적용
```

## 3. S3에서 IAM Role 액세스를 허용하는 버킷 정책 추가(원본 로그 계정에서 수행)
버킷 정책을 아래와 같이 설정합니다.
```bash
{
    "Version": "2012-10-17",
    "Statement": [
    {
        "Sid": "WAFDirectPut",
        "Effect": "Allow",
        "Principal": {
        "Service": "wafv2.amazonaws.com"
        },
        "Action": "s3:PutObject",
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1/AWSLogs/*",
        "Condition": {
        "StringEquals": {
            "s3:x-amz-acl": "bucket-owner-full-control",
            "aws:SourceAccount": [
            "{허용할 계정ID}",
            ]
        }
        }
    },
    {
        "Sid": "WAFGetBucketAcl",
        "Effect": "Allow",
        "Principal": {
        "Service": "wafv2.amazonaws.com"
        },
        "Action": "s3:GetBucketAcl",
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1",
        "Condition": {
        "StringEquals": {
            "aws:SourceAccount": [
            "{허용할 계정ID}",
            ]
        }
        }
    },
    {
        "Sid": "AWSLogDeliveryWrite",
        "Effect": "Allow",
        "Principal": {
        "Service": "delivery.logs.amazonaws.com"
        },
        "Action": "s3:PutObject",
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1/AWSLogs/*",
        "Condition": {
        "StringEquals": {
            "s3:x-amz-acl": "bucket-owner-full-control",
            "aws:SourceAccount": [
            "{허용할 계정ID}",
            ]
        }
        }
    },
    {
        "Sid": "AWSLogDeliveryAclCheck",
        "Effect": "Allow",
        "Principal": {
        "Service": "delivery.logs.amazonaws.com"
        },
        "Action": "s3:GetBucketAcl",
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1",
        "Condition": {
        "StringEquals": {
            "aws:SourceAccount": [
            "{허용할 계정ID}",
            ]
        }
        }
    },
    {
        "Sid": "CrossAccountReplicationSourceBucket",
        "Effect": "Allow",
        "Principal": {
        "AWS": "{생성한 Role ARN}"
        },
        "Action": [
        "s3:GetReplicationConfiguration",
        "s3:ListBucket"
        ],
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1"
    },
    {
        "Sid": "CrossAccountReplicationSourceObjects",
        "Effect": "Allow",
        "Principal": {
        "AWS": "{생성한 Role ARN}"
        },
        "Action": [
        "s3:GetObjectVersion",
        "s3:GetObjectVersionAcl",
        "s3:GetObjectVersionTagging"
        ],
        "Resource": "arn:aws:s3:::aws-waf-logs-cjl-collector-ue1/*"
    }
    ]
}

```

## 4. 원본 S3 버킷으로부터 로그가 복제될 대상 S3 버킷 구성(대상 로그 계정에서 수행)
버킷 정책을 아래와 같이 설정합니다.
```bash
{
    "Version": "2012-10-17",
    "Id": "PolicyForDestinationBucket",
    "Statement": [
        {
            "Sid": "Permissions on objects and buckets",
            "Effect": "Allow",
            "Principal": {
                "AWS": "{원본 계정에서 생성한 IAM Role ARN 입력}"
            },
            "Action": [
                "s3:List*",
                "s3:GetBucketVersioning",
                "s3:PutBucketVersioning",
                "s3:ReplicateDelete",
                "s3:ReplicateObject"
            ],
            "Resource": [
                "arn:aws:s3:::destination-bucket",
                "arn:aws:s3:::destination-bucket/*"
            ]
        },
        {
            "Sid": "Permission to override bucket owner",
            "Effect": "Allow",
            "Principal": {
                "AWS": "{원본 계정에서 생성한 IAM Role ARN 입력}"
            },
            "Action": "s3:ObjectOwnerOverrideToBucketOwner",
            "Resource": "arn:aws:s3:::destination-bucket/*"
        }
    ]
}

```
